# Tech spec — Edge Memo

## 1. Architecture

```txt
AVAudioEngine (foreground service)
 ↓ 16kHz mono PCM, chunked
Core ML — Whisper variant (quantized)
 ↓ transcript segments
Core ML — small instruction model
 ├──→ summary prompt → 3-paragraph summary
 └──→ actions prompt → action-item list
 ↓
On-device note store (SQLite + sqlite-vec)
 ↓ (optional, user-initiated)
E2EE sync client → self-hosted relay
```

## 2. Data shapes

```ts
interface Note {
  id: string;
  title: string;
  recordedAt: string; // ISO date
  durationSec: number;
  transcript: { startSec: number; text: string }[];
  summary: { paragraph: string }[];
  actions: { text: string; assignee: string; dueHint?: string }[];
  /** Embedding for on-device search. */
  embedding: Float32Array;
  /** Set only when the user opts into sync. */
  syncedAt?: string;
}
```

## 3. On-device pipeline

- **Transcribe**: streaming chunks of 30s, parallelized with capture so the transcript keeps pace.
- **Summarise**: triggered once on meeting-end, runs sequentially after the transcript settles.
- **Actions**: runs as the second prompt on the same model, immediately after the summary completes.

Target end-to-end latency for a 30-min meeting on iPhone 14 Pro: ≤ 4s after stop. Models are warmed at record-start so the first inference doesn't pay cold-load cost.

## 4. State management

- **Capture**: an actor owns the audio session; chunks flow to the transcriber over an `AsyncStream`.
- **Note**: assembled in memory during the meeting, persisted atomically on stop.
- **Search**: the vector index is updated on persist; queries run against sqlite-vec locally.

## 5. Error states

| Case                            | UX                                                          |
| ------------------------------- | ----------------------------------------------------------- |
| Mic permission denied           | Pre-record explainer + deep link to Settings; no capture    |
| Device below capability bar     | Offer the smaller fallback model or a download prompt       |
| Thermal pressure during capture | Inline warning at the warm threshold; auto-stop at critical |
| Summary model OOM               | Degrade to transcript-only note, flag for manual summary    |
| Sync relay unreachable          | Keep the note local, queue for the next successful sync     |
| Corrupt model bundle            | Re-download on next launch; never silently skip inference   |

## 6. Privacy & security

- **Default**: no network access on the capture or pipeline paths. Verified by a build-time entitlement check that fails CI if a networking symbol is linked under `Capture/` or `Pipeline/`.
- **Sync**: opt-in, end-to-end encrypted, user supplies their own relay URL. The server never sees plaintext.
- **Search index**: stays on-device. The vector store is per-app and encrypted at rest.
- **No analytics**: the app ships zero third-party SDKs; no telemetry leaves the device.

## 7. Test strategy

- **XCTest unit**: chunker, transcript assembler, note serializer, action-item parser.
- **XCUITest**: record → stop → review flow on the simulator.
- **Goldset scorer**: 30 recordings spanning lengths (5/15/30 min), accents (US/UK/India/AU), and content types (1:1, group, monologue), scored on:
  - WER (word error rate) — ≤ 0.12 target.
  - Summary-faithfulness rubric (Claude-as-judge over the goldset transcript).
  - Action-item recall vs. hand-annotated ground truth — ≥ 0.8 target.
- **Device matrix**: capture verified on the oldest supported chip for battery/heat.

## 8. Rollout

| Stage         | Audience       | Gate                                   |
| ------------- | -------------- | -------------------------------------- |
| Internal beta | 20 testers     | Manual QA + WER ≤ 0.12 on the goldset  |
| TestFlight    | 50 testers     | 1 week clean crash logs + flat battery |
| App Store     | Phased release | Kill-switch on the sync feature        |

## 9. Telemetry events

The product emits **nothing** off-device. The only metrics are computed locally and shown to the user (capture count, average meeting length) or produced in CI against the goldset (WER, recall, summary faithfulness). No event is ever transmitted.
