# CLAUDE.md — Edge Memo

This file is the contract between the agent and the Edge Memo codebase. Read end-to-end before touching anything non-trivial. The product is an iOS-native-first meeting-capture app where three small models cooperate locally — one transcribes, one summarises, one extracts action items. Nothing leaves the device by default; sync is opt-in and self-hosted.

## 1. Overview

Hit record, run the meeting, hit stop. Within seconds you have a transcript, a three-paragraph summary, and an action-item list — all computed on-device. The defining constraint is the privacy guarantee: the capture and inference paths never touch the network.

Brand line: _Capture on-device. Trust by default._

## 2. Tech stack

| Layer      | Choice                                      | Why                                  | Version |
| ---------- | ------------------------------------------- | ------------------------------------ | ------- |
| Platform   | iOS-native first (SwiftUI, Core ML)         | Background audio, on-device infra    | iOS 17+ |
| Transcribe | Whisper variant (Core ML, quantized)        | Strong WER for short-form speech     | latest  |
| Summarise  | Small instruction model on Core ML          | Local-only, sub-2s for a 10-min clip | latest  |
| Extract    | Same model, action-item prompt              | One artifact, two prompts            | latest  |
| Store      | SQLite + sqlite-vec (encrypted at rest)     | On-device notes + vector search      | latest  |
| Sync       | Optional, self-hosted, end-to-end encrypted | Privacy by default                   | n/a     |

## 5. Conventions

- **File names** PascalCase for Swift types; kebab-case for scripts and configs.
- **SwiftUI views stay thin** — all pipeline work lives in `Pipeline/`, never in a `View` body.
- **Prompts live in `Pipeline/Prompts/`** as versioned constants; any change ships paired with a goldset delta.
- **The capture path is offline by construction** — no networking imports are allowed under `Capture/` or `Pipeline/`.
- **One note format** — the on-disk schema is shared across iOS and Android; migrations are explicit.

## 6. Agent guardrails

✅ Free to touch: SwiftUI components, capture flows, sync UX, content.

🛑 Confirm before touching:

- **Anything that touches the network on the capture or pipeline path** — the default is offline, enforced by a build-time entitlement check.
- Model selection or quantization scheme (affects WER, latency, battery).
- Schema of the on-disk note format (existing notes must migrate cleanly).
- Signing, entitlements, `Info.plist` background modes, secrets, CI lanes.
- Destructive git operations.

## 7. Definition of done

- `swiftlint` clean; `xcodebuild test` green on the simulator.
- `./scripts/score-goldset.sh` shows no regression in WER or action-item recall versus the prior tag.
- Manual a11y pass: VoiceOver labels on capture controls, Dynamic Type respected, reduced-motion safe.
- Battery/heat sanity check on a 30-min capture on a real device.
- PR description includes a screen recording and the goldset delta table.

## 8. Domain glossary

- **Pipeline** — the three-model chain: transcribe → summarise → extract, run end-to-end on-device.
- **Goldset** — recordings with hand-graded transcripts, summaries, and action items used to score the pipeline.
- **WER** — word error rate; the transcription accuracy metric, target ≤ 0.12.
- **Sync** — the opt-in, end-to-end-encrypted relay to a user-supplied server; off by default.

## 9. Environment & secrets

There are no product API keys — the pipeline is fully local. CI-only secrets live in the runner:

```
MATCH_PASSWORD=          # fastlane match (signing)
APP_STORE_CONNECT_KEY=   # TestFlight / App Store upload
```

Never log secret values. Never check secrets in. The app ships zero analytics SDKs; if you find one in a diff, remove it.
