# CLAUDE.md — Blokz Receipt

This file is the contract between the agent and the Blokz Receipt codebase. Read end-to-end before touching anything non-trivial. The product is a mobile-first explorer that turns any on-chain transaction hash into a printable, shareable receipt.

## 1. Overview

One input field, one beautiful output. Paste a transaction hash or an explorer URL; get back a printable receipt with from / to / value / fee / status — accountant-legible, founder-shareable, screenshot-ready.

Brand line: _Receipts for the chain._

## 5. Conventions

- **File names** kebab-case (`receipt-card.tsx`). **Exports** PascalCase.
- **RSC by default**. Add `"use client"` only when motion, hooks, or browser APIs are involved. Keep client islands narrow.
- **No barrel files** — they hurt tree-shaking.
- **`cn()` helper** for conditional class composition. Never string-concat Tailwind classes.
- **Chain adapters** all implement the same `ChainAdapter` interface in `lib/chains/types.ts`. Adding a chain = adding one file.

## 6. Agent guardrails

✅ Free to touch: components, content, lib utilities, Tailwind class usage anywhere, shadcn primitives.

🛑 Confirm before touching:

- Adding or removing an npm dependency.
- `next.config.ts`, `tsconfig.json`, ESLint or Prettier configs.
- `.env*`, secrets, deploy config, GitHub Actions.
- Schema changes in Supabase (run migration locally first).
- Destructive git operations.

## 7. Definition of done

- `pnpm lint` clean.
- `pnpm typecheck` clean.
- `pnpm build` succeeds.
- Lighthouse mobile ≥ 90 on a touched route.
- Manual a11y pass: focus ring visible, keyboard nav works, reduced-motion fallback renders.
- For UI work: tested at 360 / 768 / 1440 / 1920.
- PR description includes a screenshot and links the spec section it implements.

## 8. Domain glossary

- **Hash** — a transaction identifier on a given chain.
- **Receipt** — the rendered output of a hash; the product itself.
- **Counterparty** — the address on the other end of a transaction; user-labelable.
