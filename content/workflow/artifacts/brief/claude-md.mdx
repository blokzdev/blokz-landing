# CLAUDE.md — Blokz Brief

This file is the contract between the agent and the Blokz Brief codebase. Read end-to-end before touching anything non-trivial. The product is a web-first reader that turns any arxiv link into a printable, shareable paper digest: key claims, methods, baselines, limits, plus a "why it matters" angle.

## 1. Overview

One input field, one structured output. Paste an arxiv URL or ID; get back a 5-section digest — claims, methods, baselines, limits, takeaway — exportable as PDF or share-card.

Brand line: _Papers, printed plain._

## 2. Tech stack

## 3. Commands

```bash

```

## 5. Conventions

- **File names** kebab-case (`digest-card.tsx`). **Exports** PascalCase.
- **RSC by default**. Add `"use client"` only when motion, hooks, or browser APIs are involved.
- **No barrel files** — they hurt tree-shaking.
- **`cn()` helper** for conditional class composition. Never string-concat Tailwind classes.
- **Prompts live in `lib/extract.ts`** as named const exports; they're versioned and the eval-set scorer runs on every change.

## 6. Agent guardrails

✅ Free to touch: components, content, lib utilities, Tailwind class usage anywhere, shadcn primitives.

🛑 Confirm before touching:

- Adding or removing an npm dependency.
- `next.config.ts`, `tsconfig.json`, ESLint or Prettier configs.
- `.env*`, secrets, deploy config, GitHub Actions.
- Schema changes in Supabase (run migration locally first).
- **Prompts in `lib/extract.ts`** — any change must be paired with a re-scored eval-set delta in the PR.
- Destructive git operations.

## 7. Definition of done

- `pnpm lint` clean.
- `pnpm typecheck` clean.
- `pnpm build` succeeds.
- `pnpm test:eval` shows no regression in the rubric score versus the prior tag.
- Lighthouse mobile ≥ 90 on a touched route.
- Manual a11y pass: focus ring visible, keyboard nav works, reduced-motion fallback renders.
- For UI work: tested at 360 / 768 / 1440 / 1920.
- PR description includes a screenshot and links the spec section it implements.

## 8. Domain glossary

- **Arxiv ID** — the canonical paper identifier (e.g., `2401.12345`); also accepted as a full URL.
- **Digest** — the rendered five-section output; the product itself.
- **Goldset** — the 30 hand-graded ground-truth digests used to score new extraction prompts.
